    "# Reusable narwhals pipelines with `ccflow`\n",
    "\n",
    "This notebook walks through three layers of composition for building data-frame pipelines\n",
    "on top of [narwhals](https://narwhals-dev.github.io/narwhals/), provided by `ccflow.models.narwhals`:\n",
    "\n",
    "1. **`NarwhalsFrameTransform`** — a pure `LazyFrame -> LazyFrame` step. Framework-agnostic;\n",
    "   usable standalone via `lf.pipe(transform)` without any other ccflow machinery.\n",
    "2. **`SequenceTransform`** — bundles a list of transforms into a single transform.\n",
    "3. **`NarwhalsPipelineModel`** — a `CallableModel` that wires a frame source to a list of\n",
    "   transforms, producing a new `NarwhalsFrameResult`.\n",
    "\n",
    "Plus two generic implementations:\n",
    "\n",
    "- **`JoinTransform`** — joins another callable model's frame onto the input frame.\n",
    "- **`JoinBackTransform`** — runs an inner transform on the input, joins the result back.\n",
    "\n",
    "The TPC-H benchmark is used as a worked example (Q1 → Q3)."
   ]
  },
  {
   "cell_type": "markdown",
   "id": "e99dc870",
   "metadata": {},
   "source": [
    "## Why?\n",
    "\n",
    "A typical narwhals query is just a function:\n",
    "\n",
    "```python\n",
    "def query(lineitem):\n",
    "    return (\n",
    "        lineitem.filter(...)\n",
    "        .with_columns(...)\n",
    "        .group_by(...).agg(...)\n",
    "        .sort(...)\n",
    "    )\n",
    "```\n",
    "\n",
    "This is fine for a one-shot script. But in a real codebase you usually want:\n",
    "\n",
    "- **Configurability** — the date cutoff, the segment, the group-by columns shouldn't be hard-coded.\n",
    "- **Reusability** — small pieces (e.g. \"filter by ship date\") used in many queries.\n",
    "- **Dependency injection** — swap the data source (DuckDB, parquet, mocked) without rewriting the query.\n",
    "- **Serialization** — store the entire pipeline as JSON or YAML; reload from config.\n",
    "\n",
    "The three layers below give you these properties incrementally — pay only for the level\n",
    "of structure you actually need."
   ]
  },
  {
   "cell_type": "markdown",
   "id": "a2181d8b",
   "metadata": {},
   "source": [
    "## The three layers, at a glance\n",
    "\n",
    "| Layer | What it is | What you get |\n",
    "|-------|------------|--------------|\n",
    "| `NarwhalsFrameTransform` | A pydantic model that is callable on a `LazyFrame`. | Configurable, JSON-serializable, `.pipe()`-compatible. **No ccflow runtime required.** |\n",
    "| `SequenceTransform` | A transform whose state is a list of transforms. | Composition + JSON roundtrip + nestable. Still just `.pipe()`-compatible. |\n",
    "| `NarwhalsPipelineModel` | A `CallableModel` returning `NarwhalsFrameResult`. | Sources, contexts, dependency graph, evaluators, caching — full ccflow integration. |\n",
    "\n",
    "You can stop at layer 1 if you only want better-behaved transforms in an existing codebase.\n",
    "You can stop at layer 2 if you want bundling without ccflow.\n",
    "You graduate to layer 3 when you need sourcing, configuration, or graph composition."
   ]
  },

    "### What's wrong with this picture?\n",
    "\n",
    "This works, but the function locks in:\n",
    "\n",
    "- A specific **date cutoff** (`1998-09-02`).\n",
    "- A specific **set of group-by columns**.\n",
    "- A specific **set of aggregation outputs**.\n",
    "- The expectation that the input is exactly `lineitem` from a hard-coded place.\n",
    "\n",
    "Each of those is a reasonable thing to *configure*. And while we can certainly add parameters\n",
    "and turn this into a 12-arg function, that scales poorly: it conflates \"what the pipeline does\"\n",
    "with \"where the data comes from\", and gives no good way to swap individual stages.\n",
    "\n",

    "## 4. Aside: do you *have* to subclass `NarwhalsFrameTransform`?\n",
    "\n",
    "**No.** `NarwhalsFrameTransform` is a recommended convention, not a requirement. The\n",
    "pipeline machinery accepts three flavours of \"transform\":\n",
    "\n",
    "| Flavour                      | `.pipe()` works? | Configurable params? | JSON round-trip? | Notes |\n",
    "|------------------------------|:----------------:|:--------------------:|:----------------:|-------|\n",
    "| Plain function / lambda      | ✅               | (closure only)       | ❌               | Quickest for one-offs and pipeline-local logic. |\n",
    "| Plain `ccflow.BaseModel`     | ✅               | ✅                   | ✅               | Round-trips via ccflow's `type_` discriminator. |\n",
    "| `NarwhalsFrameTransform`     | ✅               | ✅                   | ✅               | Adds a typed contract, codebase searchability, and a hook point for future cross-cutting features (validation, tracing). |\n",
    "\n",
    "The `transforms` field on both `SequenceTransform` and `NarwhalsPipelineModel` is typed as\n",
    "`Union[NarwhalsFrameTransform, Callable]`. The `NarwhalsFrameTransform` branch is what\n",
    "enables `type_`-based deserialization for any model-shaped transform; the `Callable` branch\n",
    "is the ergonomic escape hatch for plain functions (which are accepted at runtime but cannot\n",
    "be JSON serialized).\n",
    "\n",
    "**Rule of thumb:** reach for `NarwhalsFrameTransform` when the transform is reusable across\n",
    "pipelines or you want it to be discoverable in your codebase. Use a plain function for\n",
    "ad-hoc, pipeline-local logic."

    "## 6. `NarwhalsPipelineModel`: source + transforms\n",
    "\n",
    "`SequenceTransform` is still backend-agnostic — it just composes transforms. To get\n",
    "**dependency injection**, **graph composition**, and **full configuration via Hydra/JSON**,\n",
    "graduate to `NarwhalsPipelineModel`.\n",
    "\n",
    "A pipeline takes:\n",
    "\n",
    "- a **`source`**: any `CallableModel` whose `result_type` is `NarwhalsFrameResult` (or a subclass).\n",
    "- a list of **`transforms`**: applied in order via `.pipe()`.\n",
    "\n",
    "The pipeline is itself a `CallableModel` returning `NarwhalsFrameResult`. Its\n",
    "`context_type` is delegated to the source — so the pipeline transparently adopts whatever\n",
    "context type the source uses, and forwards the caller-supplied context to the source at\n",
    "call time.\n",
    "\n",
    "The output is **always lazy**: even if a transform returns an eager frame, the pipeline\n",
    "re-coerces to a `LazyFrame` so subsequent stages can rely on the lazy contract. Users that\n",
    "want an eager result call `.collect()` on the returned `NarwhalsFrameResult`.\n",
    "\n",
    "### A source for the lineitem table\n",

    "## 8. Confluence: pipelines as inputs to pipelines\n",
    "\n",
    "Now level up: each side input becomes its own `NarwhalsPipelineModel`. The \"main\" pipeline\n",
    "joins them together. This is the **confluence** pattern.\n",
    "\n",
    "```text\n",
    "   /table/customer               /table/orders             /table/lineitem\n",
    "          │                              │                          │\n",
    "          ▼                              ▼                          ▼\n",
    "   customer_pipeline             orders_pipeline            lineitem_pipeline\n",
    "   (filter segment)              (pre-filter date)          (pre-filter + derive)\n",
    "          │                              │                          │\n",
    "          └─────── JoinTransform ────────┴────── JoinTransform ─────┘\n",
    "                                         │\n",
    "                                         ▼\n",
    "                              aggregate / sort / head\n",
    "                                         │\n",
    "                                         ▼\n",
    "                                    Q3 result\n",
    "```\n",
    "\n",
    "Why bother? Now each side input is independently:\n",
    "\n",
    "- **Testable** — run `customer_pipeline()` on its own.\n",

    "- **Reusable** — drop the orders pre-filter into another query unchanged.\n",

    "## 10. When *not* to use `NarwhalsPipelineModel`\n",
    "\n",
    "The linear `.pipe()` contract is the design's strength — but it's also a constraint.\n",
    "Some queries don't fit:\n",
    "\n",
    "- **True DAGs.** TPC-H Q15 (CTE referenced from two places) and Q21 (correlated EXISTS)\n",
    "  share intermediate frames across multiple downstream consumers. Forcing them through\n",
    "  a single `.pipe()` chain duplicates work.\n",
    "- **Scalar subqueries.** Q4, Q11, Q17, Q22 compute a scalar (or a tiny frame) from one\n",
    "  query and use it as a parameter in another. The \"pipeline\" view is the wrong abstraction.\n",
    "- **Multi-output stages.** Anything that wants to fork into multiple labeled outputs\n",
    "  (e.g. `frame -> {\"by_segment\": ..., \"by_region\": ...}`) breaks the `LazyFrame -> LazyFrame`\n",
    "  contract.\n",
    "\n",
    "The right move for these is to step **up** a layer: split the query into multiple\n",
    "`CallableModel`s, compose them at the graph level, and let ccflow's evaluator handle\n",
    "parallelization and caching of shared upstream computations. The pipeline abstractions in\n",
    "this notebook live *inside* each node of that graph; they're not a replacement for it."
   ]

    "## 11. Summary\n",
    "\n",
    "Three layers, each opt-in:\n",
    "\n",
    "1. **`NarwhalsFrameTransform`** — pydantic + callable + `.pipe()`-compatible. Use anywhere\n",
    "   narwhals is used; no ccflow runtime required by callers. Subclassing is a *convention*,\n",
    "   not a requirement — see §4 for the plain-function / plain-`BaseModel` / `NarwhalsFrameTransform`\n",
    "   tradeoffs.\n",
    "2. **`SequenceTransform`** — bundle. Nestable, JSON-roundtrippable when transforms are\n",
    "   model-shaped.\n",
    "3. **`NarwhalsPipelineModel`** — full ccflow integration: sources, contexts, dependency\n",
    "   declarations, evaluator, caching, Hydra config.\n",
    "\n",
    "Plus two ready-made transforms:\n",
    "\n",
    "- **`JoinTransform`** — multi-source enrichment with another `CallableModel`.\n",
    "- **`JoinBackTransform`** — fork-and-rejoin when window functions don't fit.\n",
    "\n",
    "When pipelines compose pipelines, you get **confluence** — small, testable units glued\n",
    "together at the join boundary, each independently configurable and (in future) cacheable.\n",
    "\n",
